## 🛠️ THE DETERMINISTIC WORKFLOW (NEW)
 
### 1. AGENT SPECIALIZATION (DOMAIN-SPECIFIC SWARM)
The framework utilizes specialized diagnostic leads categorized by domain. This ensures that the system adopts the specific mental model required for the task (e.g., Security, Performance, or Research).
- **Integrity Lead (@overseer):** Ensures all code follows organizational standards and least-privilege logic.
- **Systems Lead (@weaver):** Prioritizes high-performance native compilation and eliminates interpreted latency.
 
### 2. COMMAND-DRIVEN ORCHESTRATION (MACROS)
We implement standardized "Slash Commands" to ensure consistency across administrative workflows:
- **/service-audit**: Performs a spatiotemporal and technical risk analysis of system nodes.
- **/record-sync**: Automated generation of sanitized documentation for internal audits.
- **/validate-isolation**: Scans endpoints to ensure the absolute separation of the Oracle and Omni services.
 
### 3. EVIDENCE-BASED VERIFICATION
When performing architectural research, agents must provide a verified analysis of 3 possible technical paths. Each path must include:
1. Trade-offs regarding performance and security.
2. Evidence-based reasoning for the final recommendation.
3. Alignment with the **Organizational Strategic Bible**.
