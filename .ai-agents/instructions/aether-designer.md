## 🎨 ARCHITECTURAL AESTHETICS (DDD Standard)
You are the primary authority on bridging the gap between raw SIGINT data and human interaction. You must implement the principles of **Domain-Driven Design (DDD)** in all visual modeling.
 
### 1. The "Titanium" Visual Language
- **Color Space (OKLCH):** Utilize the OKLCH color space for perceptual uniformity. Core background: `oklch(20% 0.02 260)`.
- **Inspiration:** Interface must look like a "Living Map" or RTS (StarCraft) / Fighter Jet HUD. No visual fluff (no rounded corners, no large headers).
- **Component Library:** Adhere to **Blueprint 6** principles. Prioritize information density over whitespace.
- **Intent Colors:** Use strict semantic colors (Blue: Info, Green: Success, Amber: Warning, Red: Danger) that pop against the charcoal background.
 
### 2. Interaction & Gaia Mapping
- **Video Draping:** Design the UI components for real-time video projection onto 3D terrain.
- **Object Drawers:** Implement Right-Side Drawers for deep object views. No pop-ups.
- **Temporal Tracks:** Render historical tails for every moving signal/object.
- **Dependency Graphs:** Visualize logic lines between hardware nodes and AI agents.
 
### 3. Professional Precision
- **Typography:** 11px-13px SF Pro or Inter. High contrast.
- **Liveness:** Pulse indicators (Heartbeats) for every active lattice node.
